How HTML elements are supported by screen readers

Typical HTML element support patterns by screen readers:

  • Announce roles of elements as the user moves through the content
  • Announce accessible names and/or descriptions of controls
  • Announce the text content of elements
  • Announce the start and end of elements
  • Change in voice or pitch as content is announced (e.g. to signify punctuation)
  • Announce states and properties of elements.
  • Emission of a beep or other sound when an element with a particular state or property receives virtual focus
  • Instructions on how to operate interactive elements such as form controls
  • Keyboard navigation, shortcuts and “quick access” lists showing all instances of particular elements on a page

Note: The combination of patterns supported varies from element to element. Support for (and announcements of) some elements can vary between screen reader software.

About TalkBack

TalkBack is the built-in screen reader for Android devices. It is developed by Google and is available on all Android devices.

Android TalkBack: quick gestures

To turn on one or more of the TalkBack shortcuts, go to Settings > Accessibility > TalkBack > TalkBack Shortcut

Note: Multi-finger gestures are available only with TalkBack 9.1+

  • Start/stop TalkBack:
    • Press and hold both volume buttons for 3 seconds
      (The shortcut must be enabled first)
    • Swipe up with two fingers from the bottom of the screen to turn on
      (The shortcut must be enabled first)
    • Speak "Hey Google, turn on/off TalkBack" (if supported)
  • Pause/resume announcements: Two-finger tap
  • TalkBack menu: Three-finger tap
    (Or swipe down-right / up-right in a single gesture)
  • Reading controls:
    1. Three-finger swipe in any direction
      (or swipe down-up/up-down in a single gesture)
    2. Repeat gesture to cycle through reading controls including lines, headings, links, landmarks, controls etc.
    3. Once the desired reading control is selected, swipe down/up to move to the next/previous item of that type.
  • Read next/previous item: Single-finger swipe right/left
  • Read all content: Two-finger triple-tap
  • Activate element (link, button, etc.): Single-finger double-tap (anywhere on the screen)
  • Scroll screen: Two-finger swipe up, down, left, or right
  • Search screen: Swipe left then down

Note: You can also test mobile keyboard interactions (for example, tabbing through controls) by using a bluetooth-connected keyboard.
